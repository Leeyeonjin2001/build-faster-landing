## 0. 프로젝트 핵심 관점

- **서비스**: BUILD FASTER — AI + 전문가가 함께 MVP를 1주일 안에 만들어주는 서비스
- **이 프로젝트**: 서비스를 홍보하고, 방문자를 상담 예약으로 전환하는 랜딩 페이지
- **타겟 고객**: MVP를 빠르게 만들고 싶은 창업자 / 기획자 / 사업가
- **핵심 메시지**: "당신의 문제를 1주일 안에 해결합니다" (기술 중심 ✕ → 고객 문제 중심 ✔)
- **이 서비스가 아닌 것**: 대시보드, 회원 시스템, 결제 시스템 (순수 랜딩 페이지)

### 브랜딩 핵심 원칙
- AI는 **도구**, 설계와 판단은 **전문가** — "바이브코딩 대행"이 아니라 "AI + 전문가 시너지"
- 기술이 아닌 **가치**를 보여준다 (빠른 개발 → 문제 해결)
- 논리적이고 투명한 어프로치 (재미보다 신뢰)

## 1. 가격 정책 (3단계 퍼널)

| 단계 | 이름 | 가격 | 설명 |
|------|------|------|------|
| STEP 1 | 무료 상담 + 데모 | 무료 | 30분 미팅 → 3일 내 작동하는 데모 제공 |
| STEP 2 | PRD + 실행 플랜 | 10만원 | 상세 기획서 + 기술 플랜. 가져가서 직접 개발도 가능 |
| STEP 3 | 풀 서비스 완성 | 150만원~ | 1주 내 서비스 완성 + 배포 + 프로젝트 리포트 |

### 가격표 (STEP 3 기준)

| 타입 | 가격 | 기간 |
|------|------|------|
| 컨설팅 | 10만원 | 1~2일 |
| 웹 MVP | 150만원 | 1주 |
| 앱 MVP | 200만원 | 1주 |

## 2. 고객 유형 (3가지 진입 시나리오)

| 유형 | 설명 | 예시 |
|------|------|------|
| 문제 인식형 | 문제는 알지만 어떻게 해결할지 모름 | "예약 시스템이 필요한데 어디서 시작?" |
| 시간 부족형 | 직접 할 시간이 없음 (본업이 있음) | "바이브코딩 배울 시간이 없어요" |
| 막힌 유형 | 직접 하다가 기술적으로 막힘 | "DB 연동에서 막혔어요" |
