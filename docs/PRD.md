# PRD: BUILD FASTER 랜딩 페이지

> 버전: v1.0
> 작성일: 2026-03-04

---

## 0. 프로젝트 핵심 관점

- **서비스**: BUILD FASTER — 바이브코딩으로 MVP를 빠르게 만들어주는 서비스
- **이 프로젝트**: 서비스를 홍보하는 랜딩 페이지
- **목표**: 방문자 → 상담 예약 전환
- **타겟**: MVP를 빠르게 만들고 싶은 창업자/기획자/사업가

---

## 1. 핵심 가치 제안 (Value Proposition)

### 방문자가 확인하고 싶은 2가지
1. **진짜 빠른가?** → 속도 증명 (미팅 후 3일 내 데모)
2. **AI로 퀄리티가 나오나?** → 레퍼런스, 실제 결과물

### 차별화 포인트
- 고정 가격제 (견적 받을 필요 없음)
- 미팅만 해도 데모를 빠르게 볼 수 있음 (다른 곳은 견적서만 옴)
- 바이브코딩 활용으로 압도적 속도

---

## 2. 가격 정책

| 타입 | 기간 | 가격 |
|------|------|------|
| 웹 MVP | 1주 | 150만원 |
| 앱 MVP | 1주 | 200만원 |

> 배포 포함 시 +2~3일 소요 가능

---

## 3. 랜딩 페이지 섹션 구조

### S1. 히어로 (Hero)
- 헤드라인: "아이디어를 2주 만에 작동하는 MVP로"
- 서브: 바이브코딩으로 빠르게, 합리적으로
- CTA 버튼: "무료 상담 예약하기" → 예약 섹션으로 스크롤

**통과 기준:**
- [ ] 히어로 영역이 뷰포트 전체를 채움
- [ ] CTA 클릭 → 예약 섹션으로 스무스 스크롤

### S2. 왜 BUILD FASTER? (Why Us)
- **미팅만 해도 데모를 본다** — 다른 곳은 견적서, 우리는 작동하는 데모
- **바이브코딩 = 속도** — AI 활용으로 개발 속도 극대화
- **고정 가격, 놀라운 투명성** — 웹 150만원, 앱 200만원. 끝.

**통과 기준:**
- [ ] 3개 강점이 카드/블록으로 표시됨
- [ ] 각 강점에 아이콘 + 제목 + 설명

### S3. 진행 과정 (Process / Timeline)
- Step 1: 미팅 (30분, 아이디어 공유)
- Step 2: 3일 내 데모 전달
- Step 3: 피드백 반영
- Step 4: 1주 내 완성 + 배포

**통과 기준:**
- [ ] 4단계 타임라인 시각화
- [ ] 각 단계에 기간 표시

### S4. 가격표 (Pricing)
- 웹 MVP: 150만원 / 1주
- 앱 MVP: 200만원 / 1주
- 포함사항 명시

**통과 기준:**
- [ ] 2개 가격 카드 나란히 표시
- [ ] 각 카드에 포함사항 리스트

### S5. 레퍼런스 (Portfolio)
- 만든 프로젝트 갤러리
- (현재 더미, 나중에 실제 프로젝트로 교체)

**통과 기준:**
- [ ] 그리드 레이아웃으로 프로젝트 카드 표시
- [ ] 카드: 이미지 + 프로젝트명 + 한줄 설명

### S6. 메뉴 선택 → 예약 (CTA Form)
- Step 1: 어떤 MVP? (웹 / 앱 / 기타)
- Step 2: 현재 단계? (아이디어만 / 기획서 있음 / 디자인 있음)
- Step 3: 연락처 + 희망 일정 입력
- 제출 버튼: "미팅 예약하기"

**통과 기준:**
- [ ] 3스텝 선택 폼이 순차적으로 표시
- [ ] 최종 제출 시 확인 메시지 표시

### S7. FAQ
- 자주 묻는 질문 3~5개
- (내용은 더미, 나중에 교체)

**통과 기준:**
- [ ] 아코디언 형태로 질문/답변 토글

### S8. 푸터 (Footer)
- 서비스명, 연락처, 카피라이트
